📖 اصطلاحات مهم برنامه‌نویسی

واژه‌هایی که هر برنامه‌نویسی باید بلد باشد

👋 سلام، دانشجو جان! به جلسه دوم خوش اومدی 📖

🗣️ چرا باید اصطلاحات رو بلد باشیم؟

تصور کن می‌خوای بری یه کشور خارجی، ولی هیچی از زبانشون بلد نیستی. حتی نمی‌دونی «سلام» و «مرسی» رو چطور بگی! قطعاً اذیت می‌شی و نمی‌تونی ارتباط بگیری. دنیای برنامه‌نویسی هم یه زبون داره با کلی اصطلاحات خاص خودش. اگه این واژه‌ها رو بلد باشی، می‌تونی با برنامه‌نویس‌های دیگه حرف بزنی، سوال بپرسی، جواب بگیری و سریع‌تر پیشرفت کنی.

💡 نکته طلایی: لازم نیست همین الان همه این اصطلاحات رو حفظ کنی! فقط کافیه باهاشون آشنا بشی. در طول دوره، خودبه‌خود ملکه ذهنت می‌شن.

📚 کتابخانه (Library)

کتابخانه یعنی مجموعه‌ای از کدهای آماده که دیگران نوشته‌اند و می‌تونی ازشون استفاده کنی بدون اینکه چرخ رو از اول اختراع کنی! مثلاً کتابخانه math پر از فرمول‌های ریاضی آماده است. کتابخانه random برای تولید اعداد تصادفی.

📌 مثال واقعی: کتابخونه مثل جعبه ابزار می‌مونه. به جای اینکه چکش و پیچ‌گوشتی رو خودت از صفر بسازی، می‌ری از جعبه ابزار برداری و استفاده کنی.

🏗️ فریمورک (Framework)

فریمورک یعنی اسکلت و ساختار آماده برای ساختن یه پروژه. فرقش با کتابخانه اینه که کتابخانه رو تو صدا می‌زنی و استفاده می‌کنی، ولی توی فریمورک، این فریمورکه که تو رو صدا می‌زنه!

📌 مثال واقعی: کتابخانه مثل لوازم آشپزخانه است (قابلمه، ماهیتابه، چاقو). فریمورک مثل یه آشپزخانه کامله که همه چی توش جاش مشخصه و تو فقط مواد رو می‌ذاری و غذا درست می‌کنی. جنگو (Django) و فلسک (Flask) فریمورک‌های معروف پایتون برای وب هستند.

📦 متغیر (Variable)

متغیر یعنی یه جعبه با برچسب که می‌تونی داخلش یه مقدار بذاری و هر وقت خواستی ازش استفاده کنی. اسمش رو خودت انتخاب می‌کنی!

📌 مثال واقعی: فرض کن یه کمد با کشوهای مختلف داری. روی هر کشو یه برچسب زدی: «جوراب»، «تی‌شرت»، «شلوار». هر وقت جوراب خواستی، می‌دونی کدوم کشو رو باز کنی. متغیر هم دقیقاً همینه!

🔧 تابع (Function)

تابع یعنی یه بسته کوچیک از کد که یه کار خاص انجام میده. بهش ورودی میدی، کارش رو انجام میده و خروجی تحویل میده. مثل دستگاه آبمیوه‌گیری! میوه میدی، آبمیوه تحویل می‌گیری.

📌 مثال واقعی: ماشین لباسشویی یه تابع عالیه! لباس کثیف می‌ذاری توش (ورودی)، ماشین کارش رو انجام میده (پردازش)، لباس تمیز تحویل می‌گیری (خروجی). توی پایتون تابع print() متن رو می‌گیره و روی صفحه نمایش میده.

📝 رشته یا استرینگ (String)

رشته یعنی متن! هر چیزی که داخل گیومه (دابل کوتیشن یا سینگل کوتیشن) قرار بگیری، یه رشته است. "سلام"، "Python"، "123" همگی رشته هستند.

📌 نکته: حتی اعداد هم می‌تونن رشته باشن! "123" یه رشته است، ولی 123 یه عدده. اولی رو نمی‌تونی باهاش جمع و تفریق کنی، دومی رو می‌تونی!

📁 ماژول (Module)

ماژول یعنی یه فایل پایتون که توش کد نوشتی و می‌تونی ازش توی فایل‌های دیگه استفاده کنی. هر فایل .py یه ماژوله!

📌 مثال واقعی: انگار یه دفترچه یادداشت داری که توش فرمول‌های مهم رو نوشتی. هر وقت لازم داشتی، دفترچه رو باز می‌کنی و از فرمول‌هاش استفاده می‌کنی. ماژول هم یه فایله پر از کدهای آماده.

🧱 کلاس (Class)

کلاس یعنی یه قالب یا blueprint برای ساختن اشیاء. مثل قالب کیک! قالب که کیک نیست، ولی باهاش می‌تونی کلی کیک درست کنی.

📌 مثال واقعی: یه کارخونه ماشین‌سازی رو تصور کن. کلاس مثل نقشه ساخت یه مدل ماشینه. با اون نقشه می‌تونن هزاران ماشین (آبجکت) تولید کنن. هر ماشین ویژگی‌های خودش رو داره (رنگ، سرعت، مدل) ولی همه از یه نقشه ساخته شدن.

🖥️ IDE (محیط توسعه یکپارچه)

IDE یعنی برنامه‌ای برای نوشتن کد که همه چی توش یکجا هست: ویرایشگر کد، اجراکننده، دیباگر (اشکال‌یاب)، تکمیل‌کننده خودکار کد. معروف‌ترین IDE برای پایتون PyCharm و VS Code هستند.

🐞 باگ (Bug) و دیباگ (Debug)

باگ یعنی خطا یا مشکل توی کد که باعث میشه برنامه درست کار نکنه. دیباگ یعنی پیدا کردن و رفع اون خطاها. مثل کارآگاه بازی توی کد خودت!

📌 تاریخچه جالب: اولین باگ واقعی یه پروانه (Moth) بود که توی کامپیوتر Mark II گیر کرده بود و باعث خرابی شد! از اون موقع به خطاهای برنامه‌نویسی می‌گن Bug (حشره).

🎯 خلاصه:
  • 📚 کتابخانه: جعبه ابزار آماده
  • 🏗️ فریمورک: اسکلت و ساختار پروژه
  • 📦 متغیر: جعبه برچسب‌دار برای ذخیره مقدار
  • 🔧 تابع: دستگاه آبمیوه‌گیری کد
  • 📝 رشته: متن داخل گیومه
  • 📁 ماژول: فایل پایتون قابل استفاده مجدد
  • 🧱 کلاس: قالب برای ساختن اشیاء
